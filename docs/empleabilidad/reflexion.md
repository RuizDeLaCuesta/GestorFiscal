# Reflexión Final — Proyecto Intermodular

## Qué he aprendido

He aprendido a diseñar una base de datos desde cero pensando en un caso de uso real. Antes de este proyecto, SQL era algo que usaba para hacer consultas sueltas sin entender del todo la lógica que hay detrás. Ahora entiendo por qué se normalizan las tablas, por qué los tipos de datos importan en contextos financieros, y cómo una estructura bien pensada desde el principio ahorra problemas después.

También he aprendido a trabajar con Git de forma progresiva, no como un sistema de backup donde subes todo al final, sino como una herramienta que refleja cómo has construido algo. Eso cambia cómo te organizas.

## Qué se me ha dado mejor

El diseño de la base de datos. Una vez que tuve claras las entidades y las relaciones, el resto fluía con lógica. Identificar que LINEA_FACTURA tenía que ser una tabla separada, o que CATEGORIA necesitaba un campo booleano de deducibilidad, fueron decisiones que tomé entendiendo el problema, no copiando una solución.

## Qué me ha costado más

La parte técnica del entorno de trabajo. Configurar MySQL, resolver los problemas con DBeaver, entender por qué el mismo script funciona en Workbench y no en otro cliente... son cosas que no tienen que ver con el diseño pero consumen tiempo. Me ha enseñado que el entorno importa tanto como el código.

## Qué mejoraría

El proyecto en sí es una base de datos sin aplicación. Me habría gustado tener tiempo para implementar aunque fuera una interfaz básica en Java que consumiera esas consultas y mostrara los resultados en pantalla. Sería el siguiente paso natural.

También mejoraría la documentación del diagrama E/R, añadiendo más detalle sobre las restricciones de negocio que hay detrás de cada decisión de diseño.

## Qué pasos quiero dar

A corto plazo, terminar 1º DAM con una base técnica sólida en Java y SQL. A medio plazo, empezar a trabajar con Python, R y Scala para análisis de datos financiero — es el lenguaje que usan los perfiles que me interesan y hay una comunidad enorme de recursos orientados a finanzas cuantitativas.
Además consideraré certificarme en CFA, CAIA o MIFID y complementarlo con licenciarme en Estadística, Matemáticas o ADE
A largo plazo, quiero trabajar en el cruce entre tecnología y banca privada: desarrollando herramientas de análisis, automatizando procesos de gestión patrimonial o construyendo sistemas que ayuden a tomar mejores decisiones de inversión. Este proyecto es un primer paso pequeño pero en la dirección correcta.
